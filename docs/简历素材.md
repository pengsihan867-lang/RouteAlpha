## 项目：RouteAlpha — LLM 评测与预算约束路由系统（开源）

**GitHub：** https://github.com/pengsihan867-lang/RouteAlpha  
**技术栈：** Python, XGBoost, sentence-transformers (bge-small), Gurobi, scikit-learn, RouterBench

**一句话：** 评测驱动的 predict-then-optimize 系统——先用量化级评测与校准化成功率预测刻画每个模型在每条 query 上的能力与失败模式，再用 MILP 在全局预算约束下做最优路由；强调黄金标准、无穿越回测、概率校准与诚实 baseline 对比。

> 双面叙事：评测岗看「评测体系 + 校准 + 失败归因 + LLM-judge 偏置量化」，模型策略岗看「predict-then-optimize + 成本/质量权衡」。

- **机器学习成功率预测与评测**：基于 RouterBench 搭建数据管线，并构建 48 条 golden set v1 与 eval harness；为每个候选模型训练独立 XGBoost 预测器，输入 prompt 特征并输出该模型的成功率预测。采用扩张窗口 out-of-fold 回测，评估 accuracy、AUC、Brier 和 ECE，并通过 Isotonic 校准将 overall ECE 从 0.194 降至 0.072。
- **概率校准与预算约束路由**：在每个 fold 内使用 holdout 数据做 Platt/Isotonic 校准，对比校准前后的概率质量；基于校准后的成功率预测，用 Gurobi 构建全局预算约束下的 MILP 路由模型，输出每条 query 的模型分配、真实成功率、optimality gap 和降级失败率。在 0.002/query 的固定预算下，MILP 路由真实成功率达到 0.79，使用约 always-expensive 60% 的成本达到其约 92% 的成功率，oracle 上限为 0.93。
- **LLM-as-a-judge 偏置量化**：为开放生成类任务实现 swap-and-aggregate 评测协议（正反各评一次、仅一致才采纳），量化 judge 的 position flip rate 与采纳率，识别并剔除受答案顺序影响的不可信判定，保障"用模型评模型"的标签质量。
- **量化级评测纪律（差异化）**：train / calibration / test / golden 四路数据分离，杜绝泄漏；定位并修复特征层数据穿越（TF-IDF 由全量 fit 改为按折仅在 train 上 fit）；ECE 校准前后 ablation、四条 baseline（always-cheap / always-expensive / random / oracle）诚实对比，并对失败样本做归因（降级失败率、oracle gap）。
