# 能力 Gap 与学习清单

> 配套《技术背景与设计文档》。原则：**理解（内核）系统学，工具（表层）边做边学。**
>
> 已有的不复习，这里只列**真缺**的。配合 AI 辅助，按构建节奏边做边补。

---

## A. 已经具备的（直接复用，面试要主动讲出来）

- Python、XGBoost / 特征工程
- MILP / 运筹优化（Gurobi、Pyomo 经验）
- **回测与评估方法论**（SMAPE、滚动回测、基于回测精度动态调权重）
- 不确定性 / 风险思维（量化交易背景）
- 多模型融合（XGBoost + CNN + Chronos 线性/非线性融合）

> 面试叙事：这些不是"和 AI 无关的旧技能"，而是 **eval、predict-then-optimize、数据驱动迭代** 的同构能力，只是过去作用在电价时序上。

---

## B. 必须系统补的"内核"（各约半天到一天，AI 辅助）

1. **LLM 能力边界与失败模式**
   - 模型在什么任务上系统性失败、为什么（幻觉、长上下文衰减、推理深度不足）。
   - 用途：能力预测器的物理直觉来源。

2. **LLM 评测方法论**（项目灵魂，岗位命门）
   - 评测集构建：正常 / 边界 / 对抗用例。
   - **LLM-as-judge 的坑**：位置偏置、自我偏好、冗长偏好。
   - 指标：准确率 / 通过率 / 校准 ECE。
   - **数据污染 / 泄漏**的识别与防范。

3. **概率校准**
   - Platt / Isotonic、ECE、可靠性图（reliability diagram）。
   - 缺的不是 ML 基础，是"用在 LLM 成功率预测上"这层应用。

4. **Tool use / Agent loop / ReAct**
   - agent 如何 plan–act–observe，为什么烧 token。
   - 用途：策略路由的"agent 策略"分支。

5. **成本 / 延迟机制**
   - token 计费、KV cache、reasoning token、上下文二次方成本。

## C. 项目专属要读的论文（3–5 篇，catch up 用）

- RouteLLM（arXiv 2406.18665）
- RouterBench（arXiv 2403.12031）
- RouterArena（arXiv 2510.00202）

**读法**：只抓"问题怎么建模、评测怎么做、和我的差异在哪"，不抠每个公式。

## D. 面试极可能被问的问题（提前备好答案）

- 你怎么**确认质量没掉**？怎么做归因？
- 你的评测集怎么建的？**怎么防数据污染**？
- 为什么不用贪心，要上 MILP？
- **cost per token vs cost per outcome** 的区别？你优化的是哪个？
- 预测概率**校准吗**？怎么验证？
- LLM-as-judge 有什么坑，你怎么规避？
- token 在变便宜，你这东西未来还有意义吗？（答案：Jevons 悖论 + 约束永存：单价降、总支出涨；且约束从"美元"迁移到"延迟/算力/质量"，never disappears。）
- 你改变了模型输出的哪个字段 / 哪个指标，带来了什么提升？（要有具体数字。）

B2(评测) + B5(成本)        → 动手 M1（数据管线 + 评测 harness）

## F. 关于放 GitHub 与"刷 star"

**该不该开源？** 该。公开仓库 + 高质量 README + **可复现结果** + **榜单排名**，是说服力的核心载体。

**关于"刷 star、加 tag 提曝光"——需要纠正的认知：**

- **star 是滞后的虚荣指标，是价值的"结果"而非"输入"。** 为求职刻意 game star，ROI 低，且懂行的面试官会反感（一看就是为面试包装）。
- **真正说服面试官的是**：
  1. 可复现（clone 就能跑出 README 里的数字）；
  2. 严谨的评测结果（帕累托前沿、校准、榜单名次）；
  3. 清晰的技术写作；
  4. 干净的代码；
  5. 诚实写明 limitations（反而显成熟）。
- **一个 5 star 但有硬核 benchmark 结果的仓库，吊打 500 star 的玩具。**
- **但"适度分发"是对的**：写一篇 build-in-public 的英文博客 / X thread（顺手喂出海对冲叙事）、提交 RouterArena 榜单、README 配清晰图表。这是"让好东西被看见"，不是刷数据。

**结论**：长期赌"评测严谨度 + 工程化结果"是对的；star 让它顺其自然地来，做一点真实分发帮信号传播，但绝不本末倒置。
