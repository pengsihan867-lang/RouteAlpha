# RouteAlpha 项目审查报告

> 审查日期：2026-06-29 · 范围：数据纪律、逻辑正确性、MILP 输出与评估指标

---

## 1. 数据穿越（Data Leakage）

### 1.1 已发现：TF-IDF 全量 fit（**已修复**）

**问题（修复前）：**  
`run_pipeline()` 对全量 `data.prompts` 调用 `Featurizer.transform()`。当 backend 回退到 TF-IDF 时，`TfidfVectorizer.fit_transform(prompts)` 在**包含未来测试折**的全量数据上拟合词表与 IDF 权重 → 特征层穿越。测试折的 TF-IDF 特征间接「见过」测试集词汇分布。

**影响：** 仅 TF-IDF 路径；bge-small 为冻结预训练模型、不 fit，无此问题。

**修复：**  
- `Featurizer.is_fittable`：tfidf=True，sentence_transformers=False  
- frozen 后端：全量 encode 一次（缓存）后按 fold 切片  
- fittable 后端：每 fold **仅在 train prompts 上 fit**，再 transform train/test

### 1.2 标签与模型：无穿越

- `rolling_backtest` 每 fold 只在 `train_idx` 上 `clf.fit`，在 `test_idx` 上 `predict_proba` → 正确。
- 校准（Platt/Isotonic）在 fold 内从 train 再切 `calib` 切片，不在 test 上 fit → 正确。

### 1.3 Golden set

- `golden_set.json` 为 held-out audit 集，当前 M1 管线未用于训练 → 符合设计。

## 2. 逻辑与叙事

### 2.1 「滚动回测」叫法

- RouterBench / peek.csv **无时间戳**；`shuffle: true` 后按 index 做 expanding window，实质是 **分块扩张窗口交叉验证 + out-of-fold 预测**，不是金融意义上的时序滚动。
- **建议对外表述：**「扩张窗口 out-of-fold 回测」；drift 模拟留给按 `eval_name` 留出的切片实验。

### 2.2 成本假设

- MILP 使用 `{model}|total_cost` 作为已知成本（token 数 × 单价），在路由时视为**确定性输入**。
- 合理：成本在调用前可由 prompt 长度估算；若未来加 latency 约束需扩展变量。

### 2.3 Oracle 与真实成功率

- `oracle`：若存在成功模型则选其中最便宜的 successful model → 理论上限，**不依赖预测器**。
- `realized_success_rate`：用 `y_true` 评估，**不用预测概率** → 诚实评估，正确。

## 3. MILP 求解器输出

`RoutingResult` 字段：

| 字段 | 含义 |
|------|------|
| `assignment` | DataFrame：`sample_id, model, p_success, cost, y_true` — 每条 query 路由到哪个模型 |
| `total_cost` | 路由方案总花费 |
| `expected_success` | Σ p_success（模型「自认为」的期望成功数） |
| `realized_success_rate` | assignment 中 y_true 的均值（真实成功率） |
| `status` | `optimal` / `infeasible` / `fallback` 等 |
| `extra` | `budget`, `mip_gap` 等 |

**约束：**
- ∀q：Σ_m x[q,m] = 1（每条选一个）
- Σ cost·x ≤ Budget（全局预算）

**目标：** max Σ p_success·x

## 4. 评估指标一览

### 阶段一（预测器质量）

| 指标 | 对象 | 含义 |
|------|------|------|
| accuracy | 每 model + overall | @0.5 阈值分类准确率 |
| AUC | 每 model + overall | 排序：成功样本 p 是否高于失败样本 |
| Brier | 每 model + overall | 概率 MSE |
| ECE | 每 model + overall | 校准：预测概率 vs 实际成功率 |

### 阶段二（路由决策质量）

| 指标 | 含义 |
|------|------|
| 真实成功率 | MILP/baseline 的 realized_success_rate |
| 期望成功率 | expected_success / N（模型自认，受校准影响） |
| 总成本 | 与预算对比 |
| optimality gap | oracle 真实成功率 − MILP 真实成功率（同预算或同成本下） |
| downgrade_failure_rate | 路由失败但存在其他 model 可成功的占比 |
| Pareto 前沿 | 扫预算网格：成本 vs 真实成功率曲线 |

### Baseline 对照

- always_cheap / always_expensive / random / oracle
